## Common Pitfalls

**1. Webhook signature verification** — MUST verify. Without it, anyone can forge subscription events. Use Stripe SDK helper.

**2. Idempotency** — same webhook event may arrive twice (Stripe retries). Track processed event IDs.

**3. Race condition: subscription created before user record fully synced** — handle gracefully, retry.

**4. Tax calculation complexity** — German USt 19%, EU B2B reverse charge if valid VAT ID, EU B2C with USt, non-EU varies. Use Stripe Tax to simplify (paid feature) OR manual rules.

**5. Currency** — EUR only initially. USD comes later. Don't half-ass currency support.

**6. Cancel doesn't immediately downgrade** — user keeps Pro until end of billing period. Mark `cancel_at_period_end=true`, downgrade on `customer.subscription.deleted` event.

**7. Mobile + Stripe** — Apple/Google forbid in-app subscription purchase via Stripe. Use external web checkout. Apps should explain "Subscribe at apex.com" with deep link.

**8. Refund handling** — Stripe handles refunds via Dashboard. Webhook for `charge.refunded` if you want auto-actions.

**9. Failed payment edge cases** — card expired, insufficient funds, fraud flag. Each routes differently in Stripe.

**10. Invoice format compliance** — German invoices need: name, address, USt-ID, invoice number, date, line items, taxes, totals. Stripe handles if configured correctly.
